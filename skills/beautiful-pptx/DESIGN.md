# DESIGN — beautiful-pptx 디자인 규율

발표 덱이 "프로처럼" 보이게 하는 정량 규칙 모음. 전부 코드 린트(`core.lint`)와 테마 토큰(`tokens.js`)으로 박제된다. 좌표계는 **`LAYOUT_16x9` = 가로 10 × 세로 5.625인치**. 색은 `#` 없는 6자리 HEX, 길이는 인치, 폰트는 pt. 대비비는 표준 WCAG 공식 실측값.

가장 중요한 결론 하나: **의미색 초록·주황은 흰 배경 일반 텍스트에서 WCAG AA를 못 넘는다(3.2~3.6:1).** 채움색·아이콘·큰 굵은 글자로만 쓰고, 텍스트가 필요하면 700번대로 어둡게. 이게 가드레일 1순위다.

---

## 1. 색 시스템

### 원리 — 60/30/10

프로 덱은 "중립 베이스가 화면을 지배하고, 액센트는 강조에만". 60% 베이스(흰/아주 옅은 회색) · 30% 보조(짙은 회색·네이비 = 본문·헤더) · 10% 액센트(강조 1색). **액센트를 10% 안에 가두는 게 핵심.**

### WCAG 가드레일 (실측 — 흰 배경 대비)

| 용도 | HEX | 흰 배경 대비 | AA 일반텍스트(4.5:1) |
|---|---|---|---|
| 파랑=정보 Blue-600 | `2563EB` | 5.17:1 | 통과 |
| 빨강=위험 Red-600 | `DC2626` | 4.83:1 | 통과(아슬) |
| 보라=개념 Violet-600 | `7C3AED` | 5.70:1 | 통과 |
| **초록=완료 Green-600** | `16A34A` | **3.30:1** | **실패** |
| **주황=주의 Orange-600** | `EA580C` | **3.56:1** | **실패** |
| 주황 대체 Amber-600 | `D97706` | 3.19:1 | 실패 |

→ **규칙: 초록·주황은 텍스트 색으로 쓰지 말 것.** 채움(KPI 카드 배경·막대), 18pt+ 굵은 큰 글자, 아이콘에만. 초록 텍스트가 필요하면 **Green-700 `15803D`(5.02:1)**, 주황 텍스트는 **Orange-700 `C2410C`** 로 한 단계 어둡게.

### 다크-라이트 샌드위치

표지·섹션 구분·마무리는 **다크**, 본문은 **라이트**. 다크에 임팩트, 라이트에 가독성.

**라이트 세트 (기본)**
- 베이스 60%: `FFFFFF` (또는 Slate-50 `F8FAFC`)
- 본문·제목 30%: Slate-800 `1E293B` (흰 위 14.63:1)
- 액센트 10%: Blue-600 `2563EB`
- 캡션 회색: Slate-500 `64748B` (흰 위 4.76:1 — 캡션 최소 통과)

**다크 세트 (표지·임팩트)**
- 베이스: Slate-900 `0F172A`
- 본문: Slate-300 `CBD5E1` (12.02:1 — 흰색보다 눈부심 덜)
- 제목: `FFFFFF` (17.85:1)
- 액센트: Blue-400 `60A5FA` (7.02:1 — 다크에선 한 단계 밝은 400번대)

### 의미색 — 모드별 분기

| 의미 | 라이트(흰 위) | 다크(Slate-900 위) |
|---|---|---|
| 위험 빨강 | `DC2626` | `F87171` (6.45:1) |
| 완료 초록 | `15803D` (텍스트) / `16A34A`(채움) | `4ADE80` (10.25:1) |
| 주의 주황 | `C2410C` (텍스트) / `EA580C`(채움) | `FBBF24` (10.69:1) |
| 정보 파랑 | `2563EB` | `60A5FA` (7.02:1) |
| 개념 보라 | `7C3AED` | `A78BFA` (6.56:1) |

### KPI 카드(의미색 채움 + 흰 글자)

white-on Blue-600 5.17:1 / Red-600 4.83:1 / Green-700 5.02:1 / Violet-600 5.70:1 — 통과. 단 **white-on Orange-600은 3.56:1 실패** → 주황 채움 카드엔 글자를 Slate-900 검정으로.

## 2. 타이포그래피

### 안전폰트만

pptx는 보는 사람 PC 폰트에 의존 → 시스템 기본만. Windows/Mac 양쪽에 있는 것: **Arial, Calibri, Georgia, Verdana, Tahoma, Trebuchet MS, Times New Roman**. Segoe UI는 Windows 전용 → 본문 회피, 쓰면 Calibri/Arial 폴백.

**페어링 3안(둘 다 안전폰트, 2종 이내):**
1. **Calibri / Calibri** — 굵기로만 위계. 무난·현대적. 기본 추천.
2. **Georgia(제목) / Arial(본문)** — 격식·신뢰감(컨설팅·금융).
3. **Arial Bold / Arial** — 가장 중립·범용.

### 타입 스케일

| 역할 | pt | 굵기 |
|---|---|---|
| 표지 타이틀 | 40 | Bold |
| 섹션 헤드 | 32 | Bold |
| 슬라이드 제목 | 24–28 | Bold |
| 본문 | 18–20 | Regular |
| 부가·리스트 | 16–18 | Regular |
| 캡션·출처 | 12–14 | Regular/Italic |

**바닥선: 본문 최소 18pt.** 14pt 미만은 출처·페이지번호에만. 줄간격: 본문 `lineSpacingMultiple 1.15~1.2`, 제목 `1.0`.

### 한글

안전 한글 폰트 = **맑은 고딕(Malgun Gothic)**, Mac 폴백 **Apple SD Gothic Neo**. 본문 한글은 라틴보다 1~2pt 키워야 같은 크기로 보임(한글이 더 빽빽). 텍스트에 `/[가-힣]/` 매칭되면 `fontFace`를 한글 폰트로 자동 스왑. 나눔스퀘어·Noto Sans KR은 임베드 안 하면 깨지니 피한다.

### 슬라이드당 글자수 상한

한 슬라이드 한 메시지. 본문 **40단어 / 한글 약 120~150자**, 불릿은 **최대 5줄·줄당 8단어**. 150자 초과 시 분할 경고(`L_DENSITY`).

## 3. 레이아웃·그리드 (10 × 5.625)

### 마진·세이프존

가장자리 5% 여백 = 좌우 **0.5인치**, 상하 **0.3인치**. 라이브영역 = x 0.5→9.5(폭 9.0), y 0.3→5.325.

| 영역 | x | y | w | h |
|---|---|---|---|---|
| 제목 | 0.5 | 0.35 | 9.0 | 0.8 |
| 콘텐츠 | 0.5 | 1.45 | 9.0 | 3.5 |
| 푸터(출처·페이지) | 0.5 | 5.25 | 9.0 | 0.3 |

(제목 밑 액센트 라인 좌표는 의도적으로 뺐다 — **금지**다.)

### 그리드

라이브폭 9.0인치, 거터 **0.2인치**:
- **2단 비교:** 각 폭 (9.0−0.2)/2 = 4.4인치, 좌 x=0.5 / 우 x=5.1
- **3단(KPI 3개):** 각 폭 (9.0−0.4)/3 = 2.867인치, x = 0.5 / 3.567 / 6.633
- **4단:** 각 폭 (9.0−0.6)/4 = 2.1인치

좌표를 직접 쓰지 말고 전부 `grid`/`col(n)`에서 산출 → 슬라이드 크기 바꿔도 한 줄만 고침.

### 화이트스페이스·위계

라이브영역의 **30~40%는 비운다.** 요소 사이 최소 0.2인치. 위계 4축: (1) 위치 — 위·왼쪽 먼저 읽힘, 핵심은 제목 자리 (2) 크기 — 제목 > 본문 (3) 색 — 액센트 1곳만 (4) 정렬 — 모든 요소 그리드 스냅, **좌측 정렬 기본**(가운데정렬 남발 금지).

## 4. 슬라이드 아키타입 9종 (좌표 포함)

assertion-evidence 방식: 제목이 한 문장 주장, 본문이 시각 증거(불릿보다 이해·기억 우수 — Penn State Alley).

**A. 표지 (cover)** — 다크 풀블리드 `0F172A`. 타이틀 x0.7 y2.2 w8.6 h1.2 40pt Bold White. 부제 y3.5 20pt `CBD5E1`. 메타(발표자·날짜) y5.0 14pt `64748B`.

**B. 섹션 구분 (section)** — 다크. 큰 번호 "01" x0.7 y1.6 54pt Bold `60A5FA`. 섹션명 y3.1 32pt Bold White.

**C. 핵심 메시지 (keymsg, assertion-evidence)** — 표준형. 제목(완결 한 문장) x0.5 y0.35 w9.0 h1.0 26pt Bold `1E293B`. 증거(이미지/차트/다이어그램) x0.5 y1.5 w9.0 h3.5. 출처 푸터.

**D. 핵심 수치 (kpi) — 3카드.** 카드 폭 2.867, x 0.5/3.567/6.633, y1.7 h2.3. 채움 `F8FAFC` + 상단 0.05인치 액센트 띠. 큰 숫자 40pt Bold 의미색 + 라벨 16pt `64748B` + 해석 한 줄.

**E. 차트 (chart).** 차트 좌측 2/3 x0.5 y1.5 w6.2 h3.5. "So-what" 결론 박스 우측 x6.9 w2.6 18pt — **차트는 데이터, 우측은 해석**.

**F. 비교 2단 (compare).** 좌 x0.5 w4.4 / 우 x5.1 w4.4, y1.5 h3.5. 패널 머리띠 색 구분(좌 중립, 우 액센트), 같은 행끼리 y 정렬.

**G. 인용 (quote)** — 다크 또는 옅은 회색. 큰 따옴표 글리프 x0.6 y1.0 60pt `CBD5E1`. 인용문 x1.2 y1.6 w7.6 28pt Regular. 출처 y4.3 16pt 앞에 "— ".

**H. 프로세스 (process, 3~5스텝).** 박스 n개 가로 배열 `col(n)` 좌표, y2.0 h1.2. 화살표(chevron/rightArrow) 박스 사이 거터 중앙. 번호+짧은 라벨, 액센트 1색 통일.

**I. 마무리 (closing)** — 표지와 대칭(다크). "감사합니다 / Q&A" x0.7 y2.3 40pt Bold White. 연락처 y3.6 18pt `CBD5E1`.

공통 머리(제목+푸터)는 `applyChrome(slide, theme, {title, source, page})` 하나로 빼서 모든 아키타입이 호출 → 일관성 자동 확보. `defineSlideMaster`로 TITLE(다크)·BODY(라이트)·SECTION(다크)·CLOSING(다크) 4종 등록, 푸터·페이지번호 상속.

## 5. 데이터 시각화 — Tufte 규범

data-ink 최대화 = 데이터 아닌 잉크는 다 지운다.

- **차트정크 제거:** 격자선·테두리·배경 채움·그림자·3D 다 끔.
- **pie/도넛/3D 금지.** 비율도 막대로. 화이트리스트 = `bar`·`line`·`barH`·`stackedBar`. 파이·3D 요청 와도 막대로 대체+경고.
- **직접 라벨링:** 값을 막대/선 끝에 직접(`showValue:true`). 계열 1개면 범례 제거(`showLegend:false`).
- **강조 1색:** 나머지 회색, 말하려는 막대 하나만 액센트. `chartColors:['CBD5E1','CBD5E1','2563EB','CBD5E1']`.
- **세로축 0부터**(막대). 축 라벨 차분하게 `catAxisLabelColor:'64748B'`.
- 차트 제목 끔(`showTitle:false`) — 슬라이드 제목이 대신. 우측엔 항상 "So-what" 박스 짝(아키타입 E).
- 색만으로 의미 전달 금지 → 색+마커·라벨 병행(색맹 안전).

```js
slide.addChart(pptx.ChartType.bar, data, {
  x:0.5, y:1.5, w:6.2, h:3.5,
  barDir:'col', showLegend:false, showTitle:false,
  showValue:true, dataLabelColor:'1E293B', dataLabelFontSize:12,
  valGridLine:{style:'none'}, catGridLine:{style:'none'},
  catAxisLabelColor:'64748B', valAxisLabelColor:'64748B',
  chartColors:['CBD5E1','CBD5E1','2563EB','CBD5E1'] });
```

## 6. 린트 규칙 (core.lint — 규칙 ID)

level: `fail`(반드시 고침) | `warn`(알리고 사용자 판단).

| ID | level | 조건 |
|---|---|---|
| `L_ELEMENTS` | fail | 슬라이드 최상위 시각요소 > 6 |
| `L_FONT` | fail | 본문 < 18pt 또는 제목 < 24pt |
| `L_TITLELINE` | fail | 제목 밑 액센트 라인 사용(금지) |
| `L_CONTRAST` | fail | 텍스트·배경 대비 < 4.5:1 (대형 18pt+ bold는 < 3) |
| `L_TEXTONLY` | warn | 시각요소 0 + 텍스트만 |
| `L_CENTERBODY` | warn | 본문 장문 가운데정렬 |
| `L_ACTIONTITLE` | warn | preset이 exec/pitch인데 제목이 동사 없는 명사구/콜론끝 |
| `L_DENSITY` | warn | 본문 한글 > 150자(또는 40단어) |
| `L_SOURCE` | warn | chart/kpi인데 source 비어있음 |

사용자가 색·폰트를 바꾸면 **AA 대비 자동 재검사** 후 실패하면 한 단계 어두운 값을 제안한다. 못 바꾸게 막지는 않는다.

## 7. "누구나 만족" 체크리스트

빌드 후 한 번 더 훑는 게이트.

**일관성** — 색·폰트·좌표를 단일 테마 객체에서만 참조. 슬라이드마다 즉석 색·좌표 금지. `defineSlideMaster`로 푸터·페이지번호·로고 상속.

**절제** — 슬라이드당 핵심 요소 3~5개 이내(6 초과 분할). 폰트 2종, 액센트 1색.

**접근성** — 본문 18pt+, 캡션 12pt 바닥. 모든 텍스트/배경 쌍 AA 4.5:1. 초록·주황은 텍스트로 안 씀. 색만으로 의미 전달 금지(아이콘·라벨 병기).

**아이콘·이미지** — 아이콘은 단색 라인/솔리드 한 스타일, 액센트 또는 잉크색. 사진은 풀블리드 or 그리드 정렬, 저해상도·워터마크 금지. **클립아트·이모지 장식 금지.** 이미지 위 텍스트는 어두운 오버레이(`0F172A` transparency 40) + 흰 글자.

**흔한 추함 회피 (생성기에서 차단·경고):**
1. 텍스트 꽉 찬 슬라이드(150자/40단어 초과) → 분할
2. 그라데이션·그림자·베벨 남발 → 플랫
3. 3D·파이 차트 → 막대 강제
4. 5색 이상 무지개 → 테마 색 클램프
5. 가운데정렬 장문 본문 → 좌측정렬
6. 폰트 3종 이상 → 2종 제한
7. 빨강·초록만으로 상태 구분 → 아이콘 병기
8. 클립아트·저화질 이미지·이모지 장식 → 제거
